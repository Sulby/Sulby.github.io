Introduction¶

Most people, at least in the United States, have probably heard of the Titanic. Though it sank over a century ago, the Titanic still occupies a space in today’s pop culture. It has shown up in TV, movies, and even children’s books. The story is captivating: how could one of the most overengineered ships in the world sink on its very first trip? We wanted to look deeper into this tragedy, and see what the data can tell us.

This topic is important because for engineers because extracting valuable insightful data from disasters like the Titanic can help avoid the same mistakes in the future. There are several points of failure for any vessel, but knowing if it was mechanical, human, etc. error can help to increase future reliability of transport. This topic is important for data science as well, as a disaster data set helps to further push data scicence's prediction cababilities to filter out the most impactful parameters and helps identify knowledge gaps within our current understanding of machine learning.

3. Imputation for Age Column¶

Our age column is missing a significant number of values. Obviously, we could remove the rows with missing values, but that might make us miss out on other important information that was in those rows. To help lessen the problem of missing values, we can try using an imputation technique to get estimates for what those values may have been.


Let's try using k-NN to impute missing age values. k-NN can make predictions for a point based on the point's k nearest neighbors. Our approach is based on this tutorial. First, we have to train a k-NN model using rows that we have complete data in. Then, we use our trained model to predict values for the rows where age is missing.


An advantage of using k-NN imputation is that the training data set is stored and learns from it only when it actually makes predictions, this makes it more efficient other other algorithms like SVM & Linear Regresson

In [ ]:
# knn classifier that uses the 5 nearest neighbors
knn = KNeighborsRegressor(5, weights='distance')

# get the entries that don't have an empty age
complete_entries = titanic_df[titanic_df['age'].notnull()]

# separate the complete entries into the things we are using to predict: 
# pclass, survived, sex, fare, and the things we are trying to predict: age
complete_entries_other = complete_entries[['pclass', 'survived', 'sex', 'fare', 'parch', 'sibsp']]
complete_entries_age = complete_entries[['age']]

# convert sex to number (female -> 1, male -> 0) so our KNN can use the values
complete_entries_other['sex'] = (complete_entries_other['sex'] == "female").astype(float)

# train our model using our complete data entries
fitted_model = knn.fit(complete_entries_other, complete_entries_age)

# make a DataFrame with same columns the knn was trained
# on, so we can use it as input to make predictions
titanic_trimmed = titanic_df[['pclass', 'survived', 'sex', 'fare', 'parch', 'sibsp']]
titanic_trimmed['sex'] = (titanic_trimmed['sex'] == "female").astype(float)

# make our predictions, and add them to the DataFrame
imputed_ages = fitted_model.predict(titanic_trimmed)

# pure estimates
titanic_df['imputed_age_unblended'] = imputed_ages

# real values, with estimates inserted for missing values
titanic_df['imputed_age'] = titanic_df['age']

# substitute imputed values if the real age is missing
for i, row in titanic_df.iterrows():
  if np.isnan(row['age']):
    titanic_df.at[i, 'imputed_age'] = row['imputed_age_unblended']

titanic_df = titanic_df.drop('imputed_age_unblended', axis=1)

Testing to see effectiveness of knn imputation¶

As you can see, our k-NN (or 5-NN, in this case) provided us with some values. However, you can also probably see some differences between the age and imputed_age columns. How can we tell if the imputed values are any good?


It would be nice to just find the differences between our imputed ages and the real ages. However, we can't do this. Since we would be testing with the same data we used to train, our performance results might be artificially high using this approach, which is not what we want.


Instead, we can try to answer this by setting aside the values we imputed, and testing k-NN on our original data. We split the data into two disjoint sets: the training set and the testing set. We then train our model using the training set. Once the model is trained, we can have it make predictions for items in the testing set. Since we know the correct ages for rows in our testing set, we can compare the k-NN's predictions with the actual ages, and see how well it performs. This testing approach is called holdout validation.

In [ ]:
# knn regressor
knn1 = KNeighborsRegressor(5, weights='distance')

# get the rows that aren't missing entries
complete_entries1 = titanic_df[titanic_df['age'].notnull()]
complete_entries1 = complete_entries1[['pclass', 'survived', 'sex', 'fare', 'parch', 'sibsp', 'age']]

# convert sex to number (female -> 1, male -> 0) so our KNN can use the values
complete_entries1['sex'] = (complete_entries1['sex'] == "female").astype(float)

# split the complete entries into training and testing sets
train, test = train_test_split(complete_entries1, train_size = 0.7)

# separate targets from training data; we don't want
# to use age values when trying to predict age values
train_targets = train['age']
train = train.drop('age', axis=1)

test_targets = test['age']
test = test.drop('age', axis=1)

# train our model using our complete data entries
fitted_model1 = knn1.fit(train, train_targets)

guesses = fitted_model1.predict(test)
diffs = (test_targets - guesses)
n_years = 5

print(str(len(diffs[abs(diffs) < n_years])) + " guesses out of " + str(len(test)) + " correct to within " + str(n_years) + " years")

90 guesses out of 314 correct to within 5 years

After running this test multiple times, it seems that our k-NN imputation is not amazing at estimation. Each time, we had the model predict 314 ages, and it normally got between 85 and 100 guesses correct to within 5 years. When the acceptable range is 10 years, it guesses right around half the time.


Here is a graph from one of our runs, showing how many of our guesses were accurate within different ranges.

Hypothesis Testing & Machine Learning¶

From our exploratory data analysis and visualization, we can identify four features (sex, class, age, and fare) that have an affect on the target variable (whether a person would survive or not). These are the features that will be used as the input variables for machine learning.

For our model, we will use a Random Forest Classifier because the dataset fits a supervised learning problem and we want to interpret how the different feature values influence survival yet have a more robust model than Decision Trees.

Additionally, we will use a Decision Tree Classifier in order to visualize the model and extract insight.

Questions:

  • Did older or younger passengers have a higher change of survival?
  • Did men or women have a higher chance of survival?
  • Did passengers that paid a higher or lower fare have a higher chance of survival?

Random Forest Classifier¶

Splitting Data¶

The first step is to divide the dataset into a training set and a testing set. Specifically, we will use a proportion of 70% for training and 30% for testing. This will let us train our model, then get an idea of how well it performs.

In [ ]:
# extract the output (survival) from the dataset
y = titanic_df['survived']

# extract the features from the dataset
X = titanic_df.drop(['survived', 'age', 'cabin', 'boat', 'body'], axis=1)

# convert sex to number (female -> 1, male -> 0)
X['sex'] = (X['sex'] == "female").astype(float)

# rename the imputed_age column to age
X = X.rename(columns={'imputed_age': 'age'})

# split the data into a training set and test set
X_train, X_test, y_train, y_test = train_test_split(X, y, train_size = 0.7, random_state=0)

Choosing Hyperparameters¶

The Random Forest model has a few hyperparameters, including the number of decision trees in the forest (n_estimators) and the maximum depth of the tree (max_depth), that we can adjust before training. We want to aim for high performance on both our training set and the test set and avoid overfitting. For more information about hyperparameters, see this article.

First, we will look at n_estimators. To optimize this value, we will compare the training and testing accuracies of a models using different values of the hyperparameter in isolation with the sklearn validation_curve() method. Based on plots after running multiple times, we can see that the optimal value for n_estimators is on average n_estimators=85.

In [ ]:
# a list of the hyperparameter values being tested
n_estimator_values = [10, 25, 50, 75, 85, 100, 250, 350, 500]

# compute the training and testing accuracies for each value of n_estimator
train_score, test_score = validation_curve(RandomForestClassifier(),
                                    X = X_train, 
                                    y = y_train, 
                                    param_name = 'n_estimators', 
                                    param_range = n_estimator_values, 
                                    cv = 3)

Insights¶

Passengers that paid the most fare statistically did not survive. Female passengers had the highest chance of survival. Majority of the passengers that did not survive were from 3rd class After training on the test dataset, we were able to create a decision tree where you can follow and match the properties of any passenger to see if they survive or don't with ~80% accuracy.


EDA Observations

  • more men on board, but higher percentage of women survived
  • highest survival rates in cabins B, D, E (limited data though)
  • higher rates of children survived
  • survival rates were higher among people who paid larger fares
  • survival rate was higher for higher classes (1st and 2nd)

Decision Tree Insights

  • our tree splits first on sex
    • men were much more likely not to survive
    • this supports the idea that they saved women and children first
  • men older than 9.5 predicted not to survive
  • men younger than 9.5 had higher chance of survival, number of siblings important
  • for women, class was most important
    • women in 1st and 2nd class were more likely to survive
    • for 3rd class, only young women predicted to survive
    • theory: this may have been due to location of cabins for each class
      • the Titanic sank around midnight, so most passengers may have been in their cabins
      • according to deck plans from Encyclopedia Titanica, the 1st and 2nd class cabins seem to have been located higher up in the ship. Some 3rd class cabins were located near the upper decks, and some were not.
